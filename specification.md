# 領収書OCR・スプレッドシート連携Webアプリ 仕様書

## 1. 概要
本アプリケーションは、スマートフォンやPCから領収書の画像を撮影・アップロードし、AI (Gemini) を用いて内容を自動読み取り（OCR）します。ユーザーが内容を確認・承認した後、データをGoogleスプレッドシートへ自動転記し、元の画像をGoogleドライブへ保存することで、経費精算業務を効率化します。

## 2. ユーザー体験（UX）フロー
1.  **撮影・アップロード**
    *   ユーザーはWebアプリを開き、「スキャン」ボタンからカメラを起動、または「アップロード」から画像を選択します。
2.  **OCR解析 & プレビュー**
    *   撮影された画像は即座にAIによって解析されます。
    *   解析結果（日付、店舗名、金額、明細など）が画面にプレビュー表示されます。
3.  **確認・修正・承認**
    *   ユーザーはプレビューされたデータを確認します。誤りがある場合はその場で修正します。
    *   内容に問題がなければ「承認」ボタンを押下します。
4.  **自動保存（バックグラウンド処理）**
    *   **Googleスプレッドシート**: 
        *   指定されたスプレッドシート生成先フォルダ（ID: `1EJ_P8H-lLNRE5hpEjVC4popvEZ89fDR2`）を検索します。
        *   **月次管理**: 領収書の日付に基づき、該当月のスプレッドシート（例: `YYYY-MM_経費精算`）が存在するか確認します。
        *   **新規作成/転記**:
            *   **存在する場合**: そのスプレッドシートの末尾に行を追加します。
            *   **存在しない場合**: 新規にスプレッドシートを作成し、適切なヘッダーを設定した上でデータを追加します。
    *   **Googleドライブ**: 
        *   指定された画像保存先フォルダ（ID: `1lPnc5vah9D9WSPzStdQcrbZEyAvdROSM`）に画像をアップロードします。
        *   アップロードされた画像のURLを取得し、スプレッドシートに記録します。
5.  **完了**
    *   保存完了のメッセージが表示され、ホーム画面（履歴画面）に戻ります。

## 3. 機能要件

### 3.1. クライアントサイド (フロントエンド)
*   **フレームワーク**: React (Vite) + TypeScript
*   **UIライブラリ**: Tailwind CSS
*   **基本機能**:
    *   カメラ撮影インターフェース（`react-webcam`等を利用）
    *   画像ファイル選択
    *   ローカルストレージへの履歴保存

### 3.2. AI / OCR 機能
*   **エンジン**: Google Gemini API (`gemini-1.5-flash` 推奨)
*   **抽出項目**:
    *   取引日時 (`date`)
    *   店舗名/支払先 (`vendorName`)
    *   合計金額 (`totalAmount`)
    *   通貨 (`currency`)
    *   費目カテゴリ (`category`: 食費、交通費、消耗品費など)
    *   (オプション) 明細リスト (`items`)

### 3.3. Googleサービス連携 (Backend/API)
本アプリはGoogle Cloud Platform (GCP) のAPIを利用します。
*   **認証**: Google Identity Services (OAuth 2.0)
    *   ユーザーのGoogleアカウントでログインし、SheetsおよびDriveへのアクセス権限を付与します。
*   **Google Sheets API**:
    *   **生成先フォルダID**: `1EJ_P8H-lLNRE5hpEjVC4popvEZ89fDR2`
    *   **ロジック**: 
        1.  領収書の日付から対象の年月（例: `2024-02`）を取得。
        2.  生成先フォルダ内でファイル名が `YYYY-MM` (または `YYYY-MM_経費精算`) のスプレッドシートを検索。
        3.  ヒットすればそのIDを使用。なければ新規作成。
        4.  対象シートにデータを追記 (`append`)。
*   **Google Drive API**:
    *   **画像保存フォルダID**: `1lPnc5vah9D9WSPzStdQcrbZEyAvdROSM`
    *   画像データ (JPEG/PNG) を上記フォルダへアップロード。
    *   ファイルの共有リンク（または閲覧用URL）を取得。

## 5. 開発ステップ
1.  **GCPプロジェクト設定**: Sheets/Drive APIの有効化、OAuthクライアントIDの作成。
2.  **認証機能の実装**: アプリ内に「Googleでログイン」ボタンを設置。
3.  **保存ロジックの実装**: 「承認」ボタン押下時のAPI呼び出し処理を作成。
    *   フォルダIDを指定した検索・作成ロジックの実装。
4.  **環境変数設定**: フォルダIDなどを `.env` で管理。
