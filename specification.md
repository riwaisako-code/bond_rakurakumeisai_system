# 領収書OCR・スプレッドシート連携Webアプリ 仕様書

## 1. 概要
本アプリケーションは、スマートフォンやPCから領収書の画像を撮影・アップロードし、AI (Gemini) を用いて内容を自動読み取り（OCR）します。ユーザーが内容を確認・承認した後、データをGoogleスプレッドシートへ自動転記し、元の画像をGoogleドライブへ保存することで、経費精算業務を効率化します。

## 2. ユーザー体験（UX）フロー
1.  **撮影・アップロード**
    *   ユーザーはWebアプリを開き、「スキャン」ボタンからカメラを起動、または「アップロード」から画像を選択します。
2.  **OCR解析 & プレビュー**
    *   撮影された画像は即座にAIによって解析されます。
    *   解析結果（日付、店舗名、金額、明細など）が画面にプレビュー表示されます。
3.  **確認・修正・承認**
    *   ユーザーはプレビューされたデータを確認します。誤りがある場合はその場で修正します。
    *   内容に問題がなければ「承認」ボタンを押下します。
4.  **自動保存（バックグラウンド処理）**
    *   **Googleスプレッドシート**: 指定されたシートの末尾に、解析データ（日付、店舗名、金額、費目など）が新しい行として追加されます。
    *   **Googleドライブ**: 撮影された領収書画像が指定されたフォルダに保存されます。
    *   （オプション）保存された画像のURLをスプレッドシートに記載します。
5.  **完了**
    *   保存完了のメッセージが表示され、ホーム画面（履歴画面）に戻ります。

## 3. 機能要件

### 3.1. クライアントサイド (フロントエンド)
*   **フレームワーク**: React (Vite) + TypeScript
*   **UIライブラリ**: Tailwind CSS
*   **基本機能**:
    *   カメラ撮影インターフェース（`react-webcam`等を利用）
    *   画像ファイル選択
    *   ローカルストレージへの履歴保存

### 3.2. AI / OCR 機能
*   **エンジン**: Google Gemini API (`gemini-1.5-flash` 推奨)
*   **抽出項目**:
    *   取引日時 (`date`)
    *   店舗名/支払先 (`vendorName`)
    *   合計金額 (`totalAmount`)
    *   通貨 (`currency`)
    *   費目カテゴリ (`category`: 食費、交通費、消耗品費など)
    *   (オプション) 明細リスト (`items`)

### 3.3. Googleサービス連携 (Backend/API)
本アプリはGoogle Cloud Platform (GCP) のAPIを利用します。
*   **認証**: Google Identity Services (OAuth 2.0)
    *   ユーザーのGoogleアカウントでログインし、SheetsおよびDriveへのアクセス権限を付与します。
*   **Google Sheets API**:
    *   指定されたスプレッドシートIDおよびシート名に対して、データを追記 (`append`) します。
*   **Google Drive API**:
    *   画像データ (JPEG/PNG) を指定フォルダへアップロードします。
    *   アップロード後のファイルIDまたは閲覧用URLを取得します。

| 2024/02/10 | セブンイレブン | 消耗品費 | 850 | JPY | https://drive.google.com/... |

## 5. 開発および導入ステップ
1.  **GCPプロジェクト設定**: Sheets/Drive APIの有効化、OAuthクライアントIDの作成。
2.  **認証機能の実装**: アプリ内に「Googleでログイン」ボタンを設置。
3.  **保存ロジックの実装**: 「承認」ボタン押下時のAPI呼び出し処理を作成。
4.  **環境変数設定**: スプレッドシートID、ドライブフォルダIDなどを `.env` で管理。

## 6. 既存コードとの統合
*   **現状**: `geminiService.ts` にてOCR処理は実装済み。`App.tsx` にて確認画面（詳細画面）は実装済み。
*   **変更点**:
    *   `App.tsx`: 「保存して戻る」ボタンの挙動を「スプレッドシートへ保存」に変更（または追加）。
    *   新規Service: `googleService.ts` を作成し、Sheets/Drive APIとの通信処理を集約。
